## 1. ブランドデザイン（Brand Design）

### 1-1. カラーパレット策定

**用途**: 新規ブランド立ち上げ・リブランディング時のカラーシステム構築
**業種例**: SaaS / EC / コーポレート / 営業資料

```
あなたはシニアブランドデザイナーです。以下の要件に基づき、プロフェッショナルなカラーパレットを策定してください。

## ブランド情報
- ブランド名: {{ブランド名}}
- 業種: {{SaaS / EC / コーポレート / その他}}
- ブランドパーソナリティ: {{例: 信頼性・先進性・親しみやすさ}}
- 競合他社の主要カラー: {{例: 競合A=#0066FF, 競合B=#FF6600}}

## 出力要件
以下の構成でカラーパレットを設計してください：

1. **プライマリカラー**（1色）
   - HEX, RGB, HSL値
   - 選定理由（ブランドパーソナリティとの関連）

2. **セカンダリカラー**（1〜2色）
   - プライマリとの調和根拠（補色・類似色・トライアドなど配色理論を明記）

3. **アクセントカラー**（1色）
   - CTA・重要アクションに使用する高コントラストカラー

4. **ニュートラルカラー**（5段階）
   - 背景・テキスト・ボーダー用のグレースケール
   - 50, 100, 300, 600, 900の5段階

5. **セマンティックカラー**
   - Success / Warning / Error / Info の4色

6. **アクセシビリティチェック**
   - 全テキスト×背景の組み合わせでWCAG AA以上のコントラスト比を保証
   - コントラスト比の数値を併記

## 制約
- 競合と明確に差別化できるカラーを選定すること
- デジタル（Web/アプリ）と印刷物の両方で再現性が高いこと
```

## 2. カラー設計（Color Theory）

### 2-1. 配色設計＋アクセシビリティ対応

**用途**: Webサイト・アプリ・資料の配色を設計する際
**業種例**: SaaS / EC / コーポレート / 営業資料

```
あなたはUIカラーデザインの専門家です。以下のプロジェクトの配色を設計してください。

## プロジェクト情報
- 種類: {{LP / コーポレートサイト / ECサイト / 営業資料 / Webアプリ}}
- ブランドカラー: {{プライマリ=#XXXXXX}}
- ターゲットユーザー: {{例: 30〜40代のビジネスパーソン}}
- 与えたい印象: {{例: 信頼感と先進性}}

## 出力要件

1. **メイン配色（5色以内）**
   - 各色のHEX値と使用用途
   - 配色理論の根拠（補色・分裂補色・類似色・トライアドのいずれか）

2. **UI用途別カラーマッピング**
   | 用途 | カラー | HEX | コントラスト比 |
   |------|--------|-----|----------------|
   | 背景（メイン） | | | |
   | 背景（セクション交互） | | | |
   | テキスト（見出し） | | | |
   | テキスト（本文） | | | |
   | テキスト（サブ） | | | |
   | CTA ボタン | | | |
   | CTA ボタンテキスト | | | |
   | リンク | | | |
   | ボーダー / 区切り線 | | | |

3. **コントラスト比検証**
   - テキスト×背景の全組み合わせでWCAG AA（4.5:1以上）を満たすこと
   - 大テキスト（18px以上太字 or 24px以上）はAA Large（3:1以上）でも可
   - 不合格の組み合わせがあれば代替案を提示

4. **配色の使用比率**
   - 60%: ベースカラー（背景・余白）
   - 30%: メインカラー（コンテンツ・構造）
   - 10%: アクセントカラー（CTA・強調）
```

### 2-2. ダーク/ライトモード対応配色

**用途**: ダークモード対応が必要なWebアプリ・サービスの配色設計
**業種例**: SaaS / Webアプリ

```
あなたはUIデザインの配色専門家です。以下のライトモード配色をベースに、ダークモード配色を設計してください。

## ライトモードの既存配色
- 背景: {{#FFFFFF}}
- テキスト（主）: {{#1A1A1A}}
- テキスト（副）: {{#6B7280}}
- プライマリ: {{#XXXXXX}}
- セカンダリ: {{#XXXXXX}}
- アクセント: {{#XXXXXX}}
- ボーダー: {{#E5E7EB}}
- Success: {{#10B981}}
- Warning: {{#F59E0B}}
- Error: {{#EF4444}}

## 出力要件

1. **ダークモード配色一覧**
   - ライトモードと対応する形で全色を設計
   - 背景は純黒（#000）ではなくダークグレー系を推奨（目の負担軽減）
   - プライマリカラーはダーク背景でも視認性を保つよう明度調整

2. **エレベーション（奥行き）の表現**
   - ダークモードではシャドウが効きにくいため、背景色の明度差でレイヤーを表現
   - Surface 0〜3の4段階の背景色を定義

3. **CSS変数マッピング**
   ```css
   :root { /* ライトモード */ }
   @media (prefers-color-scheme: dark) { /* ダークモード */ }
   ```
   の形式で出力

4. **注意事項**
   - 全テキスト×背景の組み合わせでWCAG AA以上を保証
   - セマンティックカラー（Success/Warning/Error）はダーク背景で彩度を下げすぎないこと
```

**用途**: ページ内の情報優先度を視覚的に整理する際
**業種例**: LP / 営業資料 / コーポレート

```
あなたはビジュアルデザインの専門家です。以下のコンテンツに対してビジュアルヒエラルキー（視覚的優先順位）を設計してください。

## ページ情報
- ページ種別: {{LP / 営業資料 / HP / 製品ページ}}

  {{箇条書きで記載。例:
  - キャッチコピー
  - サービス説明（3つの特徴）
  - 導入事例

  - FAQ
  - CTA}}

## 出力要件

   - 左揃え: 読みやすさ優先の本文・リスト
   - グリッドの崩しポイント（あれば）

5. **色と重み（Color & Weight）**
   - 各セクションのテキスト色・太さの指定
   - 視線誘導の流れ（Z型 or F型パターン）

最後に、**ワイヤーフレーム的な構成図**をテキストで示してください。
```

   - Semantic（意味）: --color-primary, --color-text, --color-bg 等
   - Component（用途）: --button-bg, --card-border 等

2. **タイポグラフィトークン**
   - --font-family-heading / --font-family-body
   - --text-xs 〜 --text-4xl（サイズスケール）
   - --leading-tight / --leading-normal / --leading-relaxed（行間）
   - --tracking-tight / --tracking-normal / --tracking-wide（字間）

3. **スペーシングトークン**
   - --space-1 〜 --space-16（4px単位のスケール）

4. **エフェクトトークン**
   - --radius-sm / --radius-md / --radius-lg / --radius-full
   - --shadow-sm / --shadow-md / --shadow-lg
   - --transition-fast / --transition-normal

5. **ブレイクポイント**
   - CSS変数ではなくメディアクエリとして定義
   - コメントで各ブレイクポイントの用途を明記

6. **ダークモード対応**
   - :root（ライト）と [data-theme="dark"]（ダーク）の切り替え構造
   - Semantic/Componentトークンのみ上書き（Primitiveは共通）

## 出力形式
- 1つのCSSファイル（design-tokens.css）として出力
- セクションごとにコメントで区切る
- 各変数に用途コメントを付記
